What We're Building

A hyperlocal grocery marketplace
for your neighborhood

Daily PayGo bridges the gap between local kirana stores and modern customers — giving stores a digital storefront and giving users the convenience of on-demand grocery ordering with instant delivery.

🛒
The Customer Problem

  • Wasted time traveling to stores for daily needs
  • Uncertain product availability until you arrive
  • Crowded stores — especially tough for elderly users
  • Fragmented ordering via WhatsApp or calls
  • No structured return or refund process

🏪
The Store Problem

  • No digital storefront or online ordering capability
  • Limited reach beyond walk-in customers
  • Losing customers to large e-commerce platforms
  • Manual and fragmented order management
  • No analytics or performance visibility
Our Solution

Give every neighborhood kirana store a digital presence. Give every customer the power to order from stores they already trust.

Authentication Design

How each user gets in

Every role has a distinct access path — designed around security, accountability, and operational control.

🧑‍💼 Customer Login Self-Register

  1. 1
    Open the app and tap Create Account
  2. 2
    Enter name, mobile number and email
  3. 3
    Accept Terms & Conditions, Privacy Policy and Consent
  4. 4
    Receive and verify a mobile OTP
  5. 5
    Account created — session persists automatically
  6. ↩
    Returning users: enter mobile → OTP → logged in

🏪 Store Agent Login Admin-Issued

  1. 1
    Admin onboards the store and creates credentials in the system
  2. 2
    Admin shares username & password with the store owner
  3. 3
    Store agent opens the Store Dashboard portal
  4. 4
    Logs in using admin-provided credentials — no self-registration
  5. 5
    Store profile visible only after admin approval
  6. ⚠
    Stores cannot sign up independently — access is controlled by Admin

🛡️ Platform Admin Internal Only

  1. 1
    Admin accesses the internal management panel
  2. 2
    Logs in with secure admin credentials
  3. 3
    Full authority to create, activate or deactivate store agents
  4. 4
    Can monitor all orders, refunds, disputes and commissions
  5. 5
    Platform-wide governance authority across all roles
  6. 🔒
    Access is strictly internal — separate from customer and store portals

Business Rules

How the platform
makes decisions

Core rules that govern platform behavior — from how orders are processed to how stores are discovered.

BR-01 · Store Discovery

Geo-radius based ranking

Stores are shown in bands: 0–5 km, 5–10 km, and 10–15 km. Within each band, stores are ranked by rating, review confidence, and reliability score.

BR-02 · Single Store Cart

One store per order

A customer can only order from one store per checkout. Adding items from a second store prompts the customer to clear their existing cart.

BR-04 · Dynamic Pricing

Price validated at checkout

Stores can change prices at any time. The system re-validates the current price at the moment of checkout — not when items are added to cart.

BR-05 · Order Acceptance SLA

60-second store response window

Once an order is placed, the store has 60 seconds to accept or reject it. No response means automatic cancellation — protecting customers from limbo.

BR-06 · Customer Cancellation

Cancel only in pending window

Customers may cancel their order only during the 60-second pending window. Once the store accepts an order, cancellation is restricted.

BR-07 · Delivery Ownership

Stores deliver — not the platform

Delivery is entirely the store's responsibility in Phase 1. The platform does not assign or manage delivery partners.

BR-09 · Returns

Structured return with evidence

Customers submit a return reason and optional photo evidence. The store handles reverse pickup. Refund is initiated only after store approval.

BR-10 · Store Access

No self-registration for stores

Stores cannot register themselves. Only the Admin can create store agent credentials — ensuring full platform control over who participates.
